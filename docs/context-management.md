# Gestion De Contexto

Este proyecto debe mantener continuidad documental durante el desarrollo de WizzyOS.

## Reglas

- Documentar cada decision tecnica relevante en `docs/memory.json`.
- Registrar avances, errores reales, pruebas y bloqueos en `docs/exec.md`.
- Actualizar `docs/plan.md` cuando cambie el objetivo, alcance o secuencia de fases.
- Mantener `docs/memory.json` como memoria de largo plazo del proyecto.
- Crear documentos, skills, tools o agentes adicionales cuando reduzcan perdida de contexto o mejoren la gestion del trabajo.
- No asumir datos de la instancia local de Home Assistant; registrar como pregunta abierta lo que falte confirmar.
- Antes de cambios importantes, revisar `docs/plan.md`, `docs/exec.md`, `docs/memory.json` y este archivo.

## Cuando Actualizar La Memoria

- Cambios de dominio, nombre, repositorio o estructura.
- Decisiones de arquitectura o instalacion.
- Errores encontrados en Home Assistant local.
- Resultados de pruebas o limitaciones del entorno.
- Nuevos pendientes, bloqueos o preguntas abiertas.
- Cambios en la entidad objetivo o en la forma de visualizacion.
